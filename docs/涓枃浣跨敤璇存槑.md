# 英语表达功能训练库：中文使用说明

## 1. 项目定位

本项目不是单纯的雅思题库，而是一套“结构优先、主题迁移”的英语写作与口语学习系统。核心范例帮助学习者掌握完整表达骨架；迁移练习要求在新主题、新人物、新情景和新观点中重组结构；补充素材用于填补练习中暴露出的具体表达缺口。

## 2. 启动方式

### Windows

双击项目根目录的 `run_windows.bat`。脚本会调用系统中的 Python 启动本地服务器。浏览器访问：

`http://localhost:8000`

### macOS / Linux

在项目根目录打开终端并执行：

`sh run_mac_linux.sh`

然后访问：

`http://localhost:8000`

### 为什么不能直接打开 index.html

网页数据位于多个 `.txt` 文件中，JavaScript 使用 `fetch()` 读取这些文件。多数浏览器会阻止 `file://` 页面读取本地文件，因此需要一个不联网的本地 HTTP 服务器。运行脚本不会把数据上传到互联网。

## 3. 页面说明

### 首页

显示课程规模、掌握进度、下一篇范例、常用提示词和推荐训练链。

### 功能覆盖

按共享、写作、口语查看功能矩阵。每张卡显示承载形式、关联范例和已掌握比例，可用于发现尚未学习的表达功能。

### 核心范例

可按场景、粒度和关键词筛选。每条范例包含：

- 完整英文与中文翻译；
- 结构拆解；
- 可跨主题复用的结构；
- 当前主题限定素材；
- 迁移方向；
- 覆盖功能编号；
- 收藏、掌握和个人笔记。

建议学习顺序：

1. 先理解全文内容和结构；
2. 不看中文复述每一部分的功能；
3. 背诵或默写完整范例；
4. 把可复用结构与主题素材区分开；
5. 进入迁移练习，用相同骨架回答新题。

### 迁移练习

练习分为三级：

- 难度1：句子替换、短答和局部结构；
- 难度2：完整段落、Part 2/Part 3回答、Task 1局部任务；
- 难度3：完整写作或较自由的口语表达。

每题可查看结构提示和参考表达。完成后按真实独立程度自评，不应仅依据答案是否与参考文本相同。

### 补充素材

素材以语块和可替换结构为主，不以孤立单词为主。可按功能、语域和关键词检索，并复制到个人笔记或新的数据文件中。

### 提示词中心

内置以下维护提示词：

- 追加核心范例；
- 追加迁移练习；
- 追加主题素材；
- 审计功能覆盖；
- 改进单条范例；
- 生成口语训练包；
- 生成写作训练包；
- 生成完整替换主题 CSS；
- 规划数据分包与 manifest；
- 根据导出进度生成弱项复习包。

点击复制后，将提示词和必要的现有记录一起发送给 AI，再把输出保存进项目即可。

### 数据与设置

可执行：

- 明暗主题切换；
- 导出学习进度；
- 导入学习进度；
- 清空本地进度；
- 复制 CSS 主题生成提示词；
- 查看数据文件结构。

## 4. 推荐训练流程

### 第一轮：建立骨架

优先学习共享范例 SH01—SH12，再学习与自己目标相关的写作或口语范例。每次只背一篇完整范例或一个功能段落。

### 第二轮：受控迁移

完成难度1和难度2练习。保留范例结构，但替换主题、人物、原因、例子、结果和评价。

### 第三轮：自由表达

完成难度3任务。先独立表达，再查看结构提示和参考答案。把真正不会表达的中文意图记录下来。

### 第四轮：补充素材

将表达缺口整理为：

- 新主题语块；
- 新动作或过程表达；
- 新描述语块；
- 新论证方式；
- 新的篇章功能。

前四类通常只需追加素材或练习；只有第五类才需要新增核心范例。

## 5. 进度保存与备份

掌握状态、收藏、练习评分和笔记保存在浏览器 `localStorage` 中，不会写回项目数据文件。清除浏览器网站数据或更换浏览器可能导致进度丢失，因此建议定期在“数据与设置”中导出进度 JSON。

导入进度时，网页会用导入内容替换当前本地进度。

## 6. 手机使用

电脑与手机处于同一局域网时，可在电脑启动服务器后，用手机访问电脑的局域网 IP，例如：

`http://192.168.1.10:8000`

需要确保系统防火墙允许 Python 的局域网访问。若只在手机本机使用，可将项目放入支持静态服务器的本地文件应用或开发工具中。

## 7. 主题替换

界面视觉集中在 `assets/css/styles.css`。提示词中心提供“生成替换主题 CSS”，可要求 AI 输出完整 CSS，然后备份旧文件并整体替换。

不要让 AI 修改 HTML 类名或 JavaScript 逻辑，否则新 CSS 可能无法与现有组件匹配。

## 8. 设计原则

- 无外部框架、字体或网络依赖；
- 数据、结构、样式和逻辑分离；
- 所有提示信息采用页面内状态组件，不使用弹窗；
- 桌面端与移动端响应式适配；
- 新增内容优先新建数据分包，而不是反复修改大型旧文件；
- 所有用户自由答案和笔记只保存在本机浏览器。
