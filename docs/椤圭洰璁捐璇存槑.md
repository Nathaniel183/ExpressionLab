# 项目设计说明

## 一、总体架构

项目采用无构建工具、无外部依赖的静态架构：

- `index.html`：页面结构与应用入口；
- `assets/css/styles.css`：全部视觉主题、布局和响应式规则；
- `assets/js/data-loader.js`：读取 manifest 与 JSONL 文本；
- `assets/js/app.js`：页面路由、筛选、复制、学习状态和数据导入导出；
- `data/*.txt`：课程内容与提示词；
- `docs/`：维护与使用文档；
- `tools/validate_data.py`：数据语法、ID和交叉引用校验。

整个网站使用单页应用方式切换界面，但不依赖任何前端框架。用户进度只存入浏览器，不改写只读课程数据。

## 二、内容模型

### 功能覆盖层

27个覆盖单元分为：

- 12个写作口语共享核心功能；
- 11个写作专属单元，包括6类Task 2、4类Task 1和现实写作；
- 4个口语专属单元，包括Part 1、Part 2、Part 3和互动修复。

每个单元记录子功能、微观能力、承载范例和迁移练习，用于审计遗漏和重复。

### 核心范例层

32个范例由以下粒度组成：

- 完整文章；
- 完整口语回答；
- 独立功能段落；
- 会话管理短片段。

每条范例将可复用结构与主题限定素材分开，并记录结构拆解、迁移方向和覆盖编号。

### 迁移练习层

42道练习分为：

- 难度1：局部替换和受控表达；
- 难度2：段落、完整回答和结构迁移；
- 难度3：自由口语、完整Task 1/Task 2及现实写作。

练习答案支持本地保存，打开提示或切换页面不会丢失。

### 补充素材层

92条素材按表达功能和语域组织，覆盖背景、定义、描述、过程、叙事、比较、因果、影响、观点、让步反驳、解决方案、预测、互动、邮件和项目汇报。

### 项目维护层

10个可复制提示词用于持续追加、修订、审计、生成训练包、替换CSS主题和生成弱项复习。

## 三、界面与交互

- 左侧导航在桌面端常驻，手机端转为抽屉；
- 所有通知使用页面顶部状态卡，不调用浏览器弹窗；
- 内容使用卡片、标签、折叠详情和双栏阅读；
- 范例可收藏、标记掌握和记录笔记；
- 练习可保存答案、自评并查看结构提示与参考表达；
- 素材和提示词支持一键复制；
- 进度支持导出、导入和二次确认清空；
- 支持浅色与深色主题；
- 对减少动态效果的系统偏好保留兼容空间。

## 四、可维护性

新增数据无需修改JS。新建相应类别的JSONL `.txt` 文件，并把文件名加入 `data/manifest.txt` 即可。分包设计允许按主题、阶段或生成批次扩充，同时保留旧版本便于回滚。

CSS以变量管理颜色、阴影、圆角、间距和字体。可通过内置提示词让AI生成一份完整替换样式，而不改变页面结构和功能。

## 五、数据安全与隐私

网站本身不联网，也不包含API密钥或外部服务。笔记、答案和学习进度位于浏览器本地存储。只有用户主动复制内容给外部AI时，相关文字才会离开本机。
